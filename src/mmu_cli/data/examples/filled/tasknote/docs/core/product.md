# Product

## Scope

- Core use case: founders track outcomes, not tasks, and connect work to decisions.
- MVP scope: projects, milestones, AI session handoff, release checklist.
- Out of scope this cycle: team collaboration and enterprise RBAC.

## User Journey

1. Acquisition: sees weekly shipping framework
2. Signup/Login: email-based auth
3. First value action: creates first milestone and marks gate status
4. Result view: sees execution dashboard and blockers
5. Return or upgrade: unlocks history depth and automation

## Feature Priority

- P0: auth, milestone board, gate checklist, session handoff
- P1: billing and paid plan limits
- P2: deeper analytics and integrations

## Definition of Done

- Functional verification: critical flows pass
- Error/empty/loading states: implemented
- Event instrumentation: key funnel events tracked
- Documentation updates: ADR and sprint notes updated
