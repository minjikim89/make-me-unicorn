# Make Me Unicorn

## 한 줄 소개
AI로 SaaS를 만들 때 생기는 실행 드리프트를 줄여주는 FounderMode 운영 시스템.

## 포지셔닝 (최종 정리)
- 카테고리: `AI-built SaaS Reliability OS`
- 타깃: 솔로 빌더/1인 SaaS 개발자
- 핵심 약속: `SaaS from scratch, without execution drift.`
- 차별점: 코드 생성이 아니라, 실행 누락(보안/결제/정책/릴리즈)을 운영적으로 막는다.

## 왜 사람들이 써야 하나
사람들이 원하는 건 "좋은 문서"가 아니라 "바로 체감되는 결과"다.
그래서 첫 경험에서 다음이 즉시 보여야 한다.

1. 내 프로젝트 상태를 한 번에 진단 (`mmu doctor`)
2. 지금 단계에서 빠진 체크포인트를 즉시 확인 (`mmu gate`)
3. LLM에 넘길 실행 컨텍스트를 즉시 생성 (`mmu start --emit bundle`)

## 제품 메시지
- Before: AI가 코드는 빨리 만들어주는데, 중요한 운영/출시 항목이 자주 빠진다.
- After: MMU가 문맥, 결정, 게이트를 고정해 누락을 줄이고 출시 확률을 높인다.

## 현재 우선순위
1. `1분 체감` 경험 강화 (`init -> doctor`)
2. 설치 장벽 제거 (PyPI 배포)
3. LLM 핸드오프 표준화 (`export`)

## 메인 원칙
- 문서는 소스 오브 트루스
- 결정은 ADR로 누적
- 게이트는 필요 시 점진적 강제
- 복잡성(그래프/플러그인)은 사용 신호 이후 도입
