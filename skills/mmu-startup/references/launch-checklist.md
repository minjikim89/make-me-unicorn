# MMU Launch Kit (Product Hunt + Social)

Reference for launch-day artifacts. Canonical source: `docs/launch/product-hunt.md` in the upstream repo.

## Product Hunt essentials

- **Tagline** (60 chars max): clear, benefit-led, no buzzwords
- **First comment**: maker story (1–2 paragraphs), why this exists, what's free
- **Tagline–description match**: the description must reinforce the tagline, not restate it
- **Topics**: 3 max, must be specific (e.g., "Developer Tools" not "Tech")
- **Gallery**: 1 hero image (1270x760), 2–3 supporting screens, optional video

## Maker story template

> I built X because I kept hitting Y. Existing tools were Z. So I tried W. Here's what I learned…

Keep first-person, concrete numbers, link to tweet thread for depth.

## Social templates

- **Twitter/X launch tweet**: "Just launched X on Product Hunt. [one-line value prop]. [link]. RT appreciated."
- **LinkedIn**: 200–300 words, frame as a builder learning, not a sales pitch. End with launch link + one ask.

## Stage progression (MMU unicorn stages)

| Stage | Score | Meaning |
|-------|-------|---------|
| Gray | 0–19 | Idea phase |
| Orange | 20–39 | Building MVP |
| Blue | 40–59 | Beta-ready |
| Purple | 60–79 | Public-launch-ready |
| Pink | 80–94 | Polished |
| Gold | 95–100 | Unicorn-grade |

Use these when explaining where the user's project is and what to ship next.
